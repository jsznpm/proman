# Gələcəkdə Kod Yazmaq Bacarığı Dəyərini İtirəcəkmi? AI Dövründə Developer Rolu Necə Dəyişir?

Son illərdə proqramlaşdırma dünyasında ən çox müzakirə olunan suallardan biri budur:

**"Əgər AI artıq kod yaza bilirsə, gələcəkdə developer-lərə ehtiyac olacaqmı?"**

Cavab sadə deyil.

Kod yazmaq yox olmayacaq, amma **kod yazmağın özü artıq əsas üstünlük olmayacaq**.

Developer rolu "kod istehsal edən insan" modelindən "sistem quran və AI-ni idarə edən mühəndis" modelinə doğru dəyişir.

---

# Kod Yazmaq Dövrünün Sonu?

Əvvəllər developer-in əsas dəyəri belə idi:

> "Mən daha sürətli və keyfiyyətli kod yaza bilirəm."

Amma AI agentləri ilə bu dəyişir.

Bu gün AI artıq:

* komponent yarada bilir
* API inteqrasiyası yaza bilir
* test yaza bilir
* refactor edə bilir
* dokumentasiya hazırlaya bilir

Yəni implementasiya prosesi sürətlə azalır.

Bir vaxtlar 2 gün çəkən iş bəzən 10 dəqiqəyə hazır olur.

Amma burada başqa problem yaranır:

**AI-nın yazdığı kod doğrudurmu?**

---

# Yazmaqdan Yoxlamağa Keçid

Gələcəyin developer-i üçün əsas bacarıq:

sadəcə kod yazmaq yox,

**kodun düzgün olub-olmadığını qiymətləndirməkdir.**

Çünki AI sürətli kod yaradır, amma:

* yanlış arxitektura seçə bilər
* təhlükəsizlik problemi yarada bilər
* performans problemi yarada bilər
* uzunmüddətli texniki borc yarada bilər

Məsələn:

AI sənə 500 sətrlik React komponent yaza bilər.

Amma sual budur:

* Bu komponent böyüyəndə işləyəcək?
* State düzgün idarə olunur?
* API strukturu doğrudur?
* Test edilə biləndir?

Bunu hələ də mühəndis qərar verir.

---

# Gələcəyin Əsas Bacarığı: Sistem Düşüncəsi

AI kod yaza bilər.

Amma AI-yə nə qurmalı olduğunu demək üçün sən sistemi anlamalısan.

Məsələn:

Bir e-commerce platforması qurursan.

AI sənə:

* React komponentləri
* Backend endpoint-ləri
* Database modelləri

yarada bilər.

Amma qərarlar:

* monolith yoxsa microservice?
* hansı cache strategiyası?
* hansı database?
* necə scale etmək?
* hansı security modeli?

bunlar mühəndislik qərarlarıdır.

---

# "Vertical Integrated Engineer" Dövrü

Gələcəyin developer-i daha çox belə olacaq:

**Frontend + Backend + Infrastructure + AI**

Yəni bir istiqamətdə dərin olmaqla yanaşı, bütün sistemi başa düşən mühəndis.

Əvvəllər:

Frontend developer:

```
UI
React
CSS
```

ilə kifayətlənə bilərdi.

Gələcəkdə:

```
Frontend
Backend
Database
Cloud
CI/CD
AI Agents
System Design
```

bilikləri daha vacib olacaq.

Bu "hər şeyi mükəmməl bilmək" deyil.

Bu, sistemi başdan sona idarə edə bilməkdir.

---

# AI ilə İşləmək Yeni Developer Skill-i Olacaq

Əvvəllər:

Developer → Kod yazırdı

İndi:

Developer → AI-yə düzgün tapşırıq verir → nəticəni yoxlayır → sistemə inteqrasiya edir

Burada ən vacib bacarıqlardan biri:

**Context engineering**

olacaq.

Yəni AI-yə düzgün kontekst vermək.

Məsələn:

Pis:

> "Mənə user dashboard yaz"

Yaxşı:

> "Bu layihənin arxitekturasına uyğun, mövcud design system istifadə edərək, React Query pattern-lərinə uyğun user dashboard komponenti hazırla"

Nəticə fərqli olacaq.

---

# AI üçün yaxşı struktur yaratmaq

Maraqlı tərəf odur ki, gələcəkdə yaxşı developer olmaq üçün kod bazasını AI üçün də rahat etmək lazım olacaq.

Məsələn:

* yaxşı folder strukturu
* düzgün naming
* design system
* monorepo strukturu
* dokumentasiya
* qaydalar

Bunlar AI-yə daha yaxşı qərar verməyə kömək edir.

---

# Bəs klassik kod yazmaq bacarığı lazımdır?

Bəli.

Çünki:

Kod oxumayan developer AI-nın səhvini də görə bilməz.

Əsas dəyişiklik budur:

Əvvəl:

```
70% kod yazmaq
30% düşünmək
```

Gələcəkdə:

```
20% kod yazmaq
80% analiz, dizayn və qərar vermək
```

modelinə yaxınlaşacaq.

---

# Nəticə

AI developer-ləri əvəz etməyəcək.

Amma AI istifadə etməyən developer-ləri, AI istifadə edən developer-lər əvəz edə bilər.

Gələcəyin güclü mühəndisi:

* daha az kod yazacaq
* daha çox sistem dizayn edəcək
* AI agentlərini idarə edəcək
* düzgün qərarlar verəcək

Yəni developer rolu yox olmur.

Sadəcə dəyişir:

**"Kod yazan insan" → "Sistem quran mühəndis"**.
